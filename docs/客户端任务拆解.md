# 客户端任务拆解

## 1. 目标

实现 Android 客户端 MVP，连接 Mac 服务端，展示远程屏幕，发送触控事件、文字输入、语音识别结果、快捷文本和快捷命令请求。

## 2. 工作目录

所有客户端代码只能写入：

- `client-android/`

共享协议只读参考：

- `docs/共享协议与MVP边界.md`

## 3. 技术要求

- Kotlin Android 原生项目。
- 使用 Gradle Wrapper，仓库不依赖全局 Gradle。
- 包名：`com.vibelink.client`
- 最低 SDK：26 或更高。
- 目标 SDK：使用本机 Android SDK 可用版本。
- UI 可使用传统 Android View，优先降低构建和依赖风险。
- 网络层可使用 JDK/Android 标准库，避免非必要外部依赖。

## 4. 功能任务

### 4.1 项目脚手架

- 创建 `settings.gradle.kts`、根 `build.gradle.kts`、`gradlew`、`gradle/wrapper/*`。
- 创建 `app` 模块。
- 创建 `AndroidManifest.xml`。
- 声明网络、麦克风、相机权限，其中相机权限可暂不使用。

### 4.2 连接配置

- 首屏提供服务端地址输入框，例如 `http://192.168.1.10:8765`。
- 提供 token 输入框。
- 提供连接按钮。
- 连接后调用 `GET /health` 验证服务端可用。
- 保存最近一次服务端地址和 token。

### 4.3 屏幕展示

- 连接成功后请求 `GET /stream?token=<token>`。
- 解析 `multipart/x-mixed-replace` MJPEG 流。
- 将 JPEG 帧解码为 Bitmap 并显示。
- 显示连接状态、最近帧时间和基础错误提示。
- 支持断开和重连。

### 4.4 触控控制

- 在远程画面上支持单击发送 `tap`。
- 支持双击发送 `doubleTap`。
- 支持拖动发送 `drag`。
- 支持双指或按钮触发滚动发送 `scroll`。
- 客户端必须将显示区域坐标转换为原始图片坐标。

### 4.5 文字输入

- 提供文本输入框。
- 支持发送文本到 `POST /api/control`，`type=text`。
- 提供“发送并回车”开关。
- 提供“复制到 Mac 剪贴板”按钮，调用 `type=clipboard`。

### 4.6 语音输入

- 使用 Android `SpeechRecognizer`。
- 提供开始/停止语音按钮。
- 识别结果写入文本输入框。
- 用户确认后发送到 Mac。
- 如果设备不支持语音识别，显示可理解错误，不影响手动文本输入。

### 4.7 快捷文本

- 调用 `GET /api/quick-texts`。
- 展示快捷文本列表。
- 点击快捷文本后填入输入框或直接发送。

### 4.8 快捷命令

- 调用 `GET /api/commands`。
- 展示命令列表。
- 点击命令后调用 `POST /api/commands/run`。
- 获取 `runId` 后定时查询 `GET /api/commands/runs/<runId>`。
- 展示状态和输出。

### 4.9 快捷按钮

- 调用 `GET /api/shortcut-buttons`。
- 展示按钮列表。
- 点击后调用 `POST /api/shortcut-buttons/run`。

## 5. UI 验收

- 启动 App 后可以输入服务端地址和 token。
- 连接成功后能看到远程画面。
- 画面区域支持点击控制。
- 底部或侧边提供文字、语音、快捷文本、快捷命令、快捷按钮入口。
- 任何网络错误都必须显示简短提示。

## 6. 构建与设备验证

必须支持：

```bash
cd client-android
./gradlew assembleDebug
./gradlew installDebug
adb -s 4e64cdf5 shell monkey -p com.vibelink.client 1
```

验证后返回：

- 修改了哪些文件。
- 构建是否通过。
- 是否安装到设备。
- 是否能启动。
- 仍未完成的风险。
