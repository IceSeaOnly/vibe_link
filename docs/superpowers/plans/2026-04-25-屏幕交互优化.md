# 屏幕交互优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 优化 VibeLink Android 客户端的屏幕固定展示、连接状态、移动端手势模式，并让 Mac 服务端支持显示器列表和单屏选择。

**Architecture:** 服务端扩展现有 HTTP JSON API，新增显示器列表、指定屏幕采集和指定屏幕坐标映射。客户端重构为顶部固定远程屏幕区 + 下方滚动功能区，使用连接状态机和 health 心跳判断断线，RemoteScreenView 内部管理“屏幕操作模式/点按模式”。

**Tech Stack:** SwiftPM + Network.framework + `screencapture`；Android Kotlin + 原生 View + HttpURLConnection + ADB 真机验证。

---

## 文件结构

### 服务端

- 修改 `server-mac/Sources/VibeLinkServerCore/Models.swift`
  - 新增 `DisplayInfo`、扩展 `HealthResponse`。
- 修改 `server-mac/Sources/VibeLinkServerCore/ScreenCapture.swift`
  - 提供显示器列表、按显示器 ID 截屏、按显示器映射坐标。
- 修改 `server-mac/Sources/VibeLinkServerCore/HTTPServer.swift`
  - `/stream?token=...&displayId=...` 传递 displayId。
- 修改 `server-mac/Sources/VibeLinkServerCore/Router.swift`
  - 新增 `GET /api/displays`。
  - `POST /api/control` 使用 control request 中的 displayId。
- 修改 `server-mac/Sources/VibeLinkServerCore/InputController.swift`
  - 坐标映射支持 displayId。
- 修改 `server-mac/Tests/VibeLinkServerCoreTests/ServerCoreTests.swift`
  - 增加显示器 API 和坐标映射相关测试。

### 客户端

- 修改 `client-android/app/src/main/java/com/vibelink/client/RemoteModels.kt`
  - 新增 `RemoteDisplay`、`InteractionMode`。
- 修改 `client-android/app/src/main/java/com/vibelink/client/ApiClient.kt`
  - 新增 `getDisplays()`、`openStream(displayId)`。
- 修改 `client-android/app/src/main/java/com/vibelink/client/RemoteScreenView.kt`
  - 移除按钮式 zoom/pan API 依赖。
  - 增加模式切换、双击缩小、单指平移、双指缩放、点按模式下点击/双击/两指右键/长按拖动。
- 修改 `client-android/app/src/main/java/com/vibelink/client/MainActivity.kt`
  - 重构布局：顶部屏幕固定且宽度打满，下方功能区滚动并保留内边距。
  - 连接配置默认折叠，单一 Connect/Disconnect 按钮。
  - health 心跳和流中断统一进入 Disconnected。
  - 显示器选择入口。

## 任务

### Task 1: 服务端显示器 API 与单屏采集

**Files:**
- Modify: `server-mac/Sources/VibeLinkServerCore/Models.swift`
- Modify: `server-mac/Sources/VibeLinkServerCore/ScreenCapture.swift`
- Modify: `server-mac/Sources/VibeLinkServerCore/HTTPServer.swift`
- Modify: `server-mac/Sources/VibeLinkServerCore/Router.swift`
- Modify: `server-mac/Sources/VibeLinkServerCore/InputController.swift`
- Test: `server-mac/Tests/VibeLinkServerCoreTests/ServerCoreTests.swift`

- [ ] 新增模型：`DisplayInfo(id: String, name: String, x: Int, y: Int, width: Int, height: Int, scale: Double, isMain: Bool)`；`HealthResponse` 增加 `displays: [DisplayInfo]`。
- [ ] `ScreenProvider.displays()` 使用 `CGGetActiveDisplayList` 枚举屏幕。
- [ ] `ScreenProvider.display(id:)` 返回匹配 display，找不到时回退主屏。
- [ ] `ScreenCapture.captureJPEG(displayId:)` 使用 `/usr/sbin/screencapture -x -t jpg -D <displayNumber> <tmp>`；displayNumber 先按 active display 列表中的 1-based 顺序映射。
- [ ] `/health` 返回当前主屏和 displays。
- [ ] 新增 `GET /api/displays`，Bearer token 鉴权。
- [ ] `/stream` 读取 query 参数 `displayId` 并采集对应屏幕。
- [ ] `ControlRequest` 增加 `displayId`，坐标映射到对应 display 的全局坐标。
- [ ] 运行 `cd server-mac && swift test`，必须通过。
- [ ] 用 curl 验证 `/api/displays` 和 `/stream?displayId=...`。

### Task 2: 客户端固定屏幕布局和连接状态机

**Files:**
- Modify: `client-android/app/src/main/java/com/vibelink/client/MainActivity.kt`
- Modify: `client-android/app/src/main/java/com/vibelink/client/ApiClient.kt`
- Modify: `client-android/app/src/main/java/com/vibelink/client/RemoteModels.kt`

- [ ] 将 Activity 根布局改为垂直 LinearLayout：顶部 `screenContainer` 固定，宽度 `MATCH_PARENT`，无左右外边距；下方 `ScrollView` 承载功能区，功能区保留 `12dp` 左右内边距。
- [ ] 连接配置区域加入 `Edit` 按钮；未保存地址/token 时展开，连接成功后折叠。
- [ ] 单一主按钮：未连接显示 `Connect`，连接中显示 `Connecting...` 且不可重复点击，已连接显示 `Disconnect`。
- [ ] 服务端流读取失败、health 连续失败、用户主动断开都调用统一 `setDisconnected(reason)`。
- [ ] 加入 health 心跳：已连接状态每 3 秒调用 `getHealth()`；连续 2 次失败断开。
- [ ] 新增显示器列表获取和选择；选择变化后重启 stream。
- [ ] 运行 `cd client-android && ./gradlew assembleDebug`，必须通过。

### Task 3: 客户端移动端手势模式

**Files:**
- Modify: `client-android/app/src/main/java/com/vibelink/client/RemoteScreenView.kt`
- Modify: `client-android/app/src/main/java/com/vibelink/client/MainActivity.kt`
- Modify: `client-android/app/src/main/java/com/vibelink/client/RemoteModels.kt`

- [ ] 新增 `InteractionMode.SCREEN` 和 `InteractionMode.POINTER`，默认 `SCREEN`。
- [ ] 在屏幕区右上角叠加一个悬浮模式按钮；`SCREEN` 显示“🖐/View”或等价短文本，`POINTER` 显示“⌖/Click”或等价短文本。
- [ ] `SCREEN` 模式：双击重置缩放到按宽度显示；单指拖动平移画面；双指开合调整缩放比例；不发送鼠标事件。
- [ ] `POINTER` 模式：单击发送 tap；双击发送 doubleTap；两指短按发送 rightClick；长按后拖动发送 drag；普通滚动按钮可保留或用两指滑动发送 scroll。
- [ ] `RemoteScreenView` 默认按宽度铺满绘制，去掉左右黑边；上下不足或超出通过 pan/zoom 处理。
- [ ] 所有发送的坐标都包含 `displayId`、`screenWidth`、`screenHeight`。
- [ ] 运行 Android 构建并用 ADB 截图确认 UI 结构。
